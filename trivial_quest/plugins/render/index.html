Module trivial_quest::plugins::render

source ·
Expand description

Author: Sidharth Rao

The render plugin handles game rendering, texturing, and cameras.

It creates a MainCamera entity and follows an entity with the CameraFollow component (usually the player). If no entity has the CameraFollow component, the camera will remain stationary. There can be at most one entity with the MainCamera component.

The plugin also loads sprites by querying for entities with the [AssetPath] component and the [Transform] component. It then replaces the [Transform] component with a [SpriteBundle] component containing the texture loaded from the [AssetPath] component and the original [Transform] component.

Schedule: - Startup: Sets up the main camera. - Update: Loads sprites. Only processes entities with the [AssetPath] component that has changed. This means that users can change the texture of an entity by changing the [AssetPath] component. - PostUpdate: Follows the entity (or no entity) with the CameraFollow component.
